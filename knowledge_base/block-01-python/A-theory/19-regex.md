# Регулярные выражения в Python

## Что такое регулярные выражения

Регулярные выражения (regex, regexp) — это мощный язык шаблонов для поиска, извлечения и замены текста по заданному правилу. Вместо того чтобы искать конкретную строку, ты описываешь **паттерн** (образец), которому должны соответствовать искомые фрагменты.

Модуль `re` встроен в Python и не требует установки.

```python
import re

text = "Мой телефон: +7 999 123-45-67, запасной: +7 999 765-43-21"
phones = re.findall(r'\+7 \d{3} \d{3}-\d{2}-\d{2}', text)
print(phones)  # ['+7 999 123-45-67', '+7 999 765-43-21']
```

## Аналогии регулярных выражений

#### 🔍 Аналогия 1: Поисковый запрос с подстановками
Regex — как поиск в Google с wildcard-символами. Ты ищешь не точную фразу, а шаблон: "все слова, начинающиеся на 'прогр'". Так `прогр*` найдёт "программа", "программист", "прогресс".

#### 🕳️ Аналогия 2: Трафарет с прорезями
Regex — как трафарет. В нём есть прорези определённой формы (паттерн). Ты прикладываешь трафарет к тексту, и через прорези "проваливаются" только подходящие фрагменты. Всё, что не совпало по форме, отсеивается.

#### 🥣 Аналогия 3: Сито для муки
Regex — как сито. Ты задаёшь размер ячеек (паттерн) и просеиваешь текст. Мелкие частицы (неподходящие символы) проходят мимо, а нужные (соответствующие шаблону) остаются в сите.

#### 🕵️ Аналогия 4: Детектив по приметам
Regex — как ориентировка детектива. Не "найти Ивана Иванова", а "найти человека: рост 170-180, волосы тёмные, куртка красная". Паттерн описывает приметы, а regex находит всех, кто подходит под описание.

#### 🎯 Аналогия 5: Шаблон для проверки пароля
Regex — как система проверки пароля. Правила: "минимум 8 символов, хотя бы одна цифра, хотя бы одна заглавная буква". Regex проверяет, соответствует ли пароль всем критериям сразу.

## Основные символы паттернов

| Символ | Значение | Пример |
|--------|----------|--------|
| `.` | Любой символ (кроме новой строки) | `a.c` → "abc", "a1c" |
| `\d` | Любая цифра (0-9) | `\d\d` → "42" |
| `\w` | Буква, цифра или `_` | `\w+` → "hello_1" |
| `\s` | Пробельный символ | `a\sb` → "a b" |
| `\D`, `\W`, `\S` | НЕ цифра, НЕ буква, НЕ пробел | |
| `*` | 0 или более повторений | `ab*c` → "ac", "abc", "abbc" |
| `+` | 1 или более повторений | `ab+c` → "abc", "abbc" |
| `?` | 0 или 1 повторение | `colou?r` → "color", "colour" |
| `{n}` | Ровно n повторений | `\d{3}` → "123" |
| `{n,m}` | От n до m повторений | `\d{2,4}` → "12", "1234" |
| `[abc]` | Любой из перечисленных | `[aeiou]` → гласные |
| `[^abc]` | Любой КРОМЕ перечисленных | `[^0-9]` → не цифра |
| `^` | Начало строки | `^Hello` |
| `$` | Конец строки | `world$` |

## Основные функции модуля re

```python
import re

text = "Цена: 100 руб, Скидка: 20 руб"

# search — найти первое совпадение
match = re.search(r'\d+', text)
print(match.group())  # 100

# findall — найти все совпадения (список)
all_numbers = re.findall(r'\d+', text)
print(all_numbers)  # ['100', '20']

# sub — заменить по шаблону
censored = re.sub(r'\d+', '***', text)
print(censored)  # Цена: *** руб, Скидка: *** руб

# match — совпадение только в начале строки
m = re.match(r'Цена', text)
print(m)  # <re.Match object>

# split — разбить строку по шаблону
parts = re.split(r'[:,]\s*', "a: 1, b: 2, c: 3")
print(parts)  # ['a', '1', 'b', '2', 'c', '3']
```

## Практические примеры

```python
# Проверка email
email_pattern = r'^[\w\.-]+@[\w\.-]+\.\w+$'
print(bool(re.match(email_pattern, "user@example.com")))  # True
print(bool(re.match(email_pattern, "invalid-email")))      # False

# Извлечение всех email из текста
text = "Пишите на info@site.ru или support@site.com"
emails = re.findall(r'[\w\.-]+@[\w\.-]+\.\w+', text)
print(emails)  # ['info@site.ru', 'support@site.com']

# Замена телефонов на заглушку
text = "Звоните: 8-800-555-35-35"
safe = re.sub(r'\d[\d-]{9,}', '[скрыто]', text)
print(safe)  # Звоните: [скрыто]
```

## Зачем это нужно

Регулярные выражения незаменимы для:
- Валидации ввода (email, телефон, пароль)
- Извлечения данных из текста (парсинг логов, HTML)
- Поиска и замены по сложным правилам
- Очистки данных от мусора

## Типичные ошибки

### Ошибка 1: Забытый raw-префикс r
```python
# ❌ Обратные слеши могут интерпретироваться неправильно
re.search('\d+', text)  # Может работать, но ненадёжно

# ✅ Всегда используй r-строки для regex
re.search(r'\d+', text)
```

### Ошибка 2: Жадные квантификаторы
```python
text = "<b>жирный</b> и <i>курсив</i>"

# ❌ Жадный .* захватит слишком много
re.findall(r'<.*>', text)  # ['<b>жирный</b> и <i>курсив</i>']

# ✅ Ленивый .*? захватит минимум
re.findall(r'<.*?>', text)  # ['<b>', '</b>', '<i>', '</i>']
```

### Ошибка 3: Экранирование спецсимволов
```python
# ❌ Точка — спецсимвол (любой символ)
re.search('a.c', 'abc')  # Совпадёт
re.search('a.c', 'aXc')  # Тоже совпадёт! (точка = любой)

# ✅ Экранируй, если нужна именно точка
re.search(r'a\.c', 'a.c')  # Только "a.c"
```

## Итого

Regex — язык шаблонов для поиска и замены текста.
Модуль `re`: `search`, `findall`, `sub`, `match`, `split`.
Основные символы: `\d` (цифра), `\w` (буква), `.` (любой), `*`, `+`, `?`, `{n,m}`, `[]`.
Всегда используй r-строки: `r'\d+'`.
Ленивые квантификаторы `*?`, `+?` захватывают минимум.
Regex мощны, но для простых задач иногда проще `in` или `.split()`.
