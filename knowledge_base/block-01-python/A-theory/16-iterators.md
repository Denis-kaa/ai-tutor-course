# Итераторы и генераторы в Python

## Что такое итератор

Итератор — это объект, который можно перебирать в цикле `for`, получая элементы по одному. Итератор "помнит" своё текущее положение и выдаёт следующий элемент по запросу.

Любой объект с методами `__iter__` и `__next__` — итератор.

```python
class Counter:
    def __init__(self, start, end):
        self.current = start
        self.end = end

    def __iter__(self):
        return self

    def __next__(self):
        if self.current >= self.end:
            raise StopIteration
        value = self.current
        self.current += 1
        return value

for n in Counter(1, 4):
    print(n)  # 1, 2, 3
```

## Что такое генератор

Генератор — это простой способ создать итератор с помощью ключевого слова `yield`. Вместо `return` функция использует `yield`, чтобы "отдать" значение и приостановиться, а при следующем обращении — продолжить с того же места.

```python
def count_up_to(n):
    i = 1
    while i <= n:
        yield i  # Отдаём значение и "засыпаем"
        i += 1

for num in count_up_to(3):
    print(num)  # 1, 2, 3
```

## Аналогии итераторов и генераторов

#### 🚰 Аналогия 1: Водопроводный кран
Генератор — как кран с водой. Вода (данные) льётся по капле, по твоему запросу (открыл кран — получил порцию). Тебе не нужно наполнять всю ванну сразу (весь список в памяти) — ты берёшь ровно столько, сколько нужно в данный момент.

#### 📺 Аналогия 2: Стриминг видео
Генератор — как Netflix. Ты не скачиваешь весь фильм (весь список) перед просмотром. Видео подгружается по кусочкам по мере просмотра. Это экономит память и позволяет начать сразу.

#### 🏭 Аналогия 3: Конвейер "по требованию"
Обычный список — как склад, где все детали уже лежат готовые (заняли место). Генератор — как конвейер, который производит деталь только тогда, когда её запросили. Склад пуст, но деталей можно сделать сколько угодно.

#### 📖 Аналогия 4: Книга, читаемая по странице
Итератор — как чтение книги по одной странице. Ты не вырываешь все страницы сразу и не раскладываешь на полу. Ты читаешь одну, переворачиваешь, читаешь следующую. Память (стол) занята только одной страницей.

#### 🎫 Аналогия 5: Очередь в банке по талонам
Итератор — как аппарат с талонами. Он выдаёт по одному талону за раз. Ты не можешь забрать все талоны сразу — аппарат выдаст следующий только по запросу. Когда талоны кончились — `StopIteration`.

## Генераторные выражения

Как list comprehension, но с круглыми скобками — создаёт генератор, а не список.

```python
# Список — занимает память целиком
squares_list = [x**2 for x in range(1000000)]  # Миллион чисел в памяти!

# Генератор — почти не занимает памяти
squares_gen = (x**2 for x in range(1000000))   # Только формула

print(sum(squares_gen))  # Считаем по мере необходимости
```

## Зачем это нужно

Генераторы экономят память. Если тебе нужно обработать миллиард строк из файла, список их все не вместит — программа упадёт с `MemoryError`. Генератор же читает по одной строке и обрабатывает на лету.

```python
# ✅ Обработка огромного файла без загрузки в память
def read_large_file(path):
    with open(path, "r", encoding="utf-8") as f:
        for line in f:
            yield line.strip()

for line in read_large_file("huge.log"):
    if "ERROR" in line:
        print(line)
```

## Типичные ошибки

### Ошибка 1: Генератор исчерпывается
```python
gen = (x for x in range(3))
print(list(gen))  # [0, 1, 2]
print(list(gen))  # [] — генератор уже "пуст"!

# ✅ Если нужно пройти дважды — создай заново или сохрани в список
```

### Ошибка 2: yield вместо return в цикле
```python
# ❌ return прервёт функцию после первого элемента
def bad():
    for i in range(3):
        return i  # Вернёт только 0

# ✅ yield отдаёт по одному и продолжает
def good():
    for i in range(3):
        yield i  # Отдаст 0, 1, 2
```

## Итого

Итератор — объект с `__iter__` и `__next__`, перебирается в `for`.
Генератор — функция с `yield`, простой способ создать итератор.
Генераторные выражения — `(x for x in ...)` вместо `[x for x in ...]`.
Главное преимущество — экономия памяти (ленивые вычисления).
Генератор исчерпывается после одного прохода.
Идеальны для больших файлов и бесконечных последовательностей.
