# Функции в Python

## Что такое функция

Функция — это именованный блок кода, который выполняет определённую задачу. Ты определяешь функцию один раз, а потом вызываешь её сколько угодно раз.

Функции позволяют:
- Не повторять один и тот же код (принцип DRY — Don't Repeat Yourself)
- Разбивать сложную программу на маленькие понятные части
- Легче поддерживать и тестировать код

Базовый синтаксис:
```python
def greet(name):
    return f"Привет, {name}!"

message = greet("Алексей")
print(message)  # Привет, Алексей!
```

## Аналогии функции

#### 🥤 Аналогия 1: Блендер

Функция — это блендер. Ты кидаешь внутрь ингредиенты (аргументы), нажимаешь кнопку (вызываешь функцию) — и получаешь сок (результат через return). Один и тот же блендер можно использовать много раз с разными фруктами.

```python
def make_juice(fruit):
    return f"{fruit} сок"

print(make_juice("яблоко"))   # яблоко сок
print(make_juice("апельсин")) # апельсин сок
```

#### 🍳 Аналогия 2: Рецепт блюда

Функция — это записанный рецепт. Ты один раз описал, как приготовить омлет, и теперь можешь готовить его снова и снова, просто меняя количество яиц.

#### 🏭 Аналогия 3: Станок на заводе

Функция — это станок. На вход подаётся сырьё (аргументы), станок делает свою работу (тело функции), на выходе — готовая деталь (return).

#### 📐 Аналогия 4: Математическая формула

Функция — как формула. Формула площади прямоугольника: S = a * b. Ты подставляешь разные значения a и b, и каждый раз получаешь правильный результат.

#### 🎭 Аналогия 5: Актёр с ролью

Функция — как актёр, который играет роль. Ты говоришь ему, кого играть (аргументы), он выходит на сцену (выполнение), и зрители видят результат (return).

## Аргументы функции

### Позиционные и именованные аргументы

```python
def greet(name, greeting="Привет"):
    return f"{greeting}, {name}!"

print(greet("Алексей"))            # Привет, Алексей!
print(greet("Мария", "Добрый день")) # Добрый день, Мария!
```

### *args и **kwargs

```python
# *args — произвольное количество позиционных аргументов
def sum_all(*args):
    return sum(args)

print(sum_all(1, 2, 3))     # 6

# **kwargs — произвольное количество именованных аргументов
def print_info(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")

print_info(name="Алексей", age=30)
```

## Возврат значений (return)

```python
def get_greeting(name):
    return f"Привет, {name}!"

result = get_greeting("Алексей")
print(result)  # Привет, Алексей!
```

**Аналогия:** return — как курьер, который приносит результат. print() — как когда ты кричишь из окна (все слышат, но никто не может использовать). return — как когда ты передаёшь посылку в руки.

## Область видимости (scope)

```python
x = 10  # глобальная переменная

def my_function():
    y = 20  # локальная переменная
    print(x)  # 10 — может читать глобальную
    print(y)  # 20

my_function()
# print(y)  # Ошибка! y существует только внутри функции
```

**Аналогия:** Область видимости — как комнаты в доме. Глобальная переменная — это предмет в коридоре (виден отовсюду). Локальная переменная — предмет в закрытой комнате (виден только внутри).

## Итого

Функция — именованный блок кода для выполнения задачи.

**Определение:** def имя(аргументы): ... return результат
**Типы аргументов:** позиционные, именованные (со значением по умолчанию), *args, **kwargs
**return** — возвращает значение. Без return функция возвращает None.
**Область видимости:** локальные переменные видны только внутри функции, глобальные — везде.
Главный принцип: одна функция — одна задача.
