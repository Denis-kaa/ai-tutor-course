# Наследование классов в Python

## Что такое наследование

Наследование — это механизм, позволяющий создать новый класс на основе существующего. Новый класс (потомок, дочерний) получает все атрибуты и методы родителя, и может добавить свои или изменить унаследованные.

Это главный инструмент для переиспользования кода в ООП: не пиши одно и то же дважды.

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return "..."

class Dog(Animal):  # Dog наследует от Animal
    def speak(self):  # Переопределяем метод
        return f"{self.name} говорит: Гав!"

class Cat(Animal):
    def speak(self):
        return f"{self.name} говорит: Мяу!"

dog = Dog("Шарик")
print(dog.speak())  # Шарик говорит: Гав!
print(dog.name)     # Шарик (унаследовано от Animal)
```

## Аналогии наследования

#### 👨‍👩‍👧 Аналогия 1: Семья — родители и дети
Наследование — как биологическое родство. Ребёнок наследует от родителей цвет глаз, группу крови, черты характера (атрибуты и методы). Но при этом у ребёнка могут появиться свои уникальные черты (новые методы) или он может вести себя иначе, чем родители (переопределение методов).

#### 🚗 Аналогия 2: Базовая модель авто и комплектации
Класс-родитель — это базовая комплектация автомобиля (есть двигатель, колёса, руль). Класс-потомок — это та же машина, но в комплектации "люкс": добавлен кондиционер, кожаный салон, автопилот. База та же, но функционал расширен.

```python
class Car:
    def __init__(self, brand):
        self.brand = brand
    def drive(self):
        return "Еду"

class LuxuryCar(Car):
    def open_sunroof(self):  # Новый метод
        return "Люк открыт"
```

#### 📄 Аналогия 3: Шаблон документа
Родительский класс — как шаблон договора с общими пунктами. Дочерний класс — это заполненный договор для конкретного клиента: общие пункты на месте, но добавлены специфические условия.

#### 🧬 Аналогия 4: Эволюция видов
Наследование — как эволюция. Новый вид происходит от предка, сохраняя базовые черты (позвоночник, сердце), но приобретает новые адаптации (крылья, жабры). Предок не исчезает — он продолжает существовать параллельно.

#### 🏗️ Аналогия 5: Надстройка этажей
Родительский класс — это фундамент и первые этажи здания. Дочерний класс — это надстроенные сверху этажи. Ты не строишь фундамент заново, а используешь готовый и добавляешь своё.

## Функция super()

`super()` позволяет обратиться к методу родительского класса. Чаще всего используется в `__init__`, чтобы не дублировать код инициализации.

```python
class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.salary = salary

class Manager(Employee):
    def __init__(self, name, salary, team_size):
        super().__init__(name, salary)  # Вызываем __init__ родителя
        self.team_size = team_size      # Добавляем своё

m = Manager("Иван", 100000, 5)
print(m.name)       # Иван (от родителя)
print(m.team_size)  # 5 (своё)
```

## Переопределение методов (Override)

Дочерний класс может заменить поведение метода родителя своим собственным.

```python
class Bird:
    def move(self):
        return "Иду пешком"

class Eagle(Bird):
    def move(self):  # Переопределяем
        return "Лечу в небе"

class Penguin(Bird):
    def move(self):  # Переопределяем по-своему
        return "Плыву в воде"
```

## Проверка наследования

```python
print(isinstance(m, Manager))   # True
print(isinstance(m, Employee))  # True (Manager — потомок Employee)
print(issubclass(Manager, Employee))  # True
```

## Зачем это нужно

Без наследования пришлось бы копировать одинаковый код в каждый похожий класс. Наследование создаёт иерархию: общий код — в родителе, специфика — в потомках. Это делает код короче, понятнее и легче в поддержке.

## Типичные ошибки

### Ошибка 1: Забыли вызвать super().__init__
```python
# ❌ Атрибуты родителя не инициализированы
class Manager(Employee):
    def __init__(self, name, salary, team_size):
        self.team_size = team_size
        # name и salary не установлены!

m = Manager("Иван", 100000, 5)
print(m.name)  # AttributeError!

# ✅ Вызываем super().__init__
class Manager(Employee):
    def __init__(self, name, salary, team_size):
        super().__init__(name, salary)
        self.team_size = team_size
```

### Ошибка 2: Слишком глубокая иерархия
```python
# ❌ 10 уровней наследования — кошмар для понимания
# ✅ Держи иерархию плоской: 2-3 уровня максимум
```

## Итого

Наследование — создание класса на основе другого: `class Child(Parent):`.
Потомок получает все атрибуты и методы родителя.
`super().__init__()` — вызов конструктора родителя.
Переопределение (override) — замена метода родителя своим.
`isinstance()` и `issubclass()` — проверка принадлежности.
Не злоупотребляй глубиной иерархии.
