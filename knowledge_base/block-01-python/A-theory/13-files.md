# Работа с файлами в Python

## Что такое работа с файлами

Программы часто должны сохранять данные между запусками (настройки, логи, результаты) или читать их из внешних источников. Для этого Python предоставляет встроенные функции для работы с файловой системой.

Основные операции:
- **Чтение** — получить данные из файла
- **Запись** — сохранить данные в файл
- **Дозапись** — добавить данные в конец файла

```python
# Простейший пример записи и чтения
with open("notes.txt", "w", encoding="utf-8") as f:
    f.write("Привет, файл!")

with open("notes.txt", "r", encoding="utf-8") as f:
    content = f.read()
    print(content)  # Привет, файл!
```

## Аналогии работы с файлами

#### 📂 Аналогия 1: Папка с документами на столе
Файл — как документ в папке. Ты можешь открыть папку (open), прочитать документ (read), дописать в него новую страницу (write/append), а потом закрыть и положить обратно (close). Контекстный менеджер `with` — как секретарь, который сам закрывает папку, даже если ты отвлёкся.

#### 📖 Аналогия 2: Книга с закладкой
Чтение файла — как чтение книги. У тебя есть закладка (курсор), которая показывает, где ты остановился. `read()` читает всю книгу сразу. `readline()` читает по одной странице. `readlines()` вырывает все страницы и кладёт их в стопку (список).

```python
with open("book.txt", "r") as f:
    first_page = f.readline()  # Одна строка
    all_pages = f.readlines()  # Все строки списком
```

#### 📝 Аналогия 3: Блокнот для записей
Режимы открытия файла — как разные блокноты:
- `"r"` — блокнот только для чтения (нельзя писать)
- `"w"` — чистый блокнот (старые записи стираются)
- `"a"` — блокнот для дозаписи (пишешь в конец, старое остаётся)

```python
# Режим "w" — перезаписывает файл
with open("log.txt", "w") as f:
    f.write("Новая запись\n")

# Режим "a" — добавляет в конец
with open("log.txt", "a") as f:
    f.write("Ещё одна запись\n")
```

#### 🗄️ Аналогия 4: Архивный шкаф
Файловая система — как архивный шкаф. У каждого документа есть своё место (путь). Ты можешь искать документы по имени, создавать новые папки, перекладывать бумаги. Модуль `os` и `pathlib` — это твои помощники-архивариусы.

```python
import os
print(os.listdir("."))  # Список файлов в текущей папке
print(os.path.exists("notes.txt"))  # Существует ли файл?
```

#### 📬 Аналогия 5: Почтовый ящик
Запись в файл — как отправка письма в почтовый ящик. Ты кладёшь письмо (данные) в ящик (файл). Если ящик пустой — письмо первое. Если уже есть письма — твоё добавляется сверху (режим "a") или заменяет все старые (режим "w").

## Контекстный менеджер with

Конструкция `with` гарантирует, что файл будет закрыт автоматически, даже если внутри блока произойдёт ошибка.

```python
# ❌ Плохо: можно забыть закрыть файл
f = open("data.txt", "r")
data = f.read()
f.close()  # А если ошибка выше? Файл останется открытым!

# ✅ Хорошо: файл закроется автоматически
with open("data.txt", "r") as f:
    data = f.read()
# Здесь файл уже закрыт
```

## Работа с JSON

JSON — стандартный формат для обмена данными. Python преобразует словари в JSON и обратно.

```python
import json

data = {"name": "Алексей", "age": 30, "skills": ["Python", "SQL"]}

# Запись в файл
with open("user.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)

# Чтение из файла
with open("user.json", "r", encoding="utf-8") as f:
    loaded_data = json.load(f)
    print(loaded_data["name"])  # Алексей
```

## Работа с CSV

CSV — табличный формат (значения через запятую).

```python
import csv

# Запись
with open("users.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.writer(f)
    writer.writerow(["Имя", "Возраст"])
    writer.writerow(["Алексей", 30])

# Чтение
with open("users.csv", "r", encoding="utf-8") as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)  # ['Имя', 'Возраст'], ['Алексей', '30']
```

## Зачем это нужно

Без работы с файлами:
- Данные терялись бы при закрытии программы
- Невозможно было бы обмениваться данными между программами
- Не было бы логов, конфигураций, сохранений
- Программы были бы "одноразовыми"

## Типичные ошибки

### Ошибка 1: FileNotFoundError
```python
# ❌ Файл не существует
with open("missing.txt", "r") as f:
    data = f.read()  # FileNotFoundError!

# ✅ Проверка существования или обработка ошибки
import os
if os.path.exists("missing.txt"):
    with open("missing.txt", "r") as f:
        data = f.read()
else:
    print("Файл не найден")
```

### Ошибка 2: Забытая кодировка
```python
# ❌ На Windows может быть ошибка с кириллицей
with open("russian.txt", "r") as f:
    text = f.read()  # UnicodeDecodeError!

# ✅ Всегда указывай encoding="utf-8"
with open("russian.txt", "r", encoding="utf-8") as f:
    text = f.read()
```

### Ошибка 3: Режим "w" стирает данные
```python
# ❌ Осторожно! "w" удаляет старое содержимое
with open("important.txt", "w") as f:
    f.write("Новые данные")  # Старые данные удалены!

# ✅ Если нужно добавить — используй "a"
with open("important.txt", "a") as f:
    f.write("\nДополнительные данные")
```

## Итого

Файлы открываются через `open(path, mode, encoding)`.
Режимы: `"r"` (чтение), `"w"` (запись), `"a"` (дозапись).
Контекстный менеджер `with` гарантирует закрытие файла.
JSON: `json.dump()` / `json.load()`.
CSV: `csv.writer()` / `csv.reader()`.
Всегда указывай `encoding="utf-8"` для текстовых файлов.
