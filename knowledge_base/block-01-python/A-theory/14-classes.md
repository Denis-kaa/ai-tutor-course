# Классы и ООП в Python

## Что такое класс и объект

**Класс** — это чертёж или шаблон, описывающий, какие данные (атрибуты) и действия (методы) будут у объектов.
**Объект** — это конкретный экземпляр, созданный по этому чертежу.

ООП (объектно-ориентированное программирование) — это подход, где программа строится из взаимодействующих объектов, а не просто из последовательности команд.

```python
class Dog:
    def __init__(self, name, breed):
        self.name = name    # Атрибут
        self.breed = breed  # Атрибут

    def bark(self):         # Метод
        return f"{self.name} говорит: Гав!"

# Создаём объекты
dog1 = Dog("Шарик", "Дворняжка")
dog2 = Dog("Рекс", "Овчарка")

print(dog1.bark())  # Шарик говорит: Гав!
print(dog2.name)    # Рекс
```

## Аналогии классов и объектов

#### 🏗️ Аналогия 1: Чертёж дома и сам дом
Класс — это архитектурный чертёж. На нём указано: сколько комнат, где окна, какая крыша. Объект — это реальный дом, построенный по этому чертежу. По одному чертежу можно построить 100 разных домов (объектов), каждый со своим адресом и цветом стен.

```python
class House:
    def __init__(self, address, color):
        self.address = address
        self.color = color

house1 = House("ул. Ленина 5", "красный")
house2 = House("ул. Мира 12", "синий")
```

#### 🍪 Аналогия 2: Форма для печенья и печенье
Класс — это металлическая форма для печенья (звёздочка, сердечко). Объекты — это сами печенья, выдавленные из этой формы. Все печенья одной формы, но каждое можно украсить по-своему (разные атрибуты).

#### 🚗 Аналогия 3: Модель автомобиля и конкретные машины
Класс `Car` описывает, что у машины есть марка, цвет, скорость, и она может ехать и тормозить. Объекты — это конкретные машины: красная Toyota, синий Ford. У каждой свои характеристики, но ведут они себя одинаково.

```python
class Car:
    def __init__(self, brand, color):
        self.brand = brand
        self.color = color
        self.speed = 0

    def accelerate(self, amount):
        self.speed += amount
        return f"{self.brand} разгоняется до {self.speed} км/ч"

my_car = Car("Toyota", "красный")
print(my_car.accelerate(50))  # Toyota разгоняется до 50 км/ч
```

#### 👤 Аналогия 4: Анкета человека и заполненные анкеты
Класс — это бланк анкеты с полями: Имя, Возраст, Город. Объект — это заполненная анкета конкретного человека. У всех анкет одинаковая структура, но разные данные.

#### 🎭 Аналогия 5: Роль в пьесе и актёры
Класс — это роль в пьесе (Гамлет). В роли прописано, что персонаж должен говорить и делать. Объекты — это актёры, играющие эту роль. Каждый актёр играет по-своему, но следует одному сценарию.

## Ключевые концепции

### __init__ и self

`__init__` — это конструктор, метод, который вызывается автоматически при создании объекта.
`self` — это ссылка на сам объект. Через `self` объект обращается к своим атрибутам и методам.

```python
class Person:
    def __init__(self, name, age):
        self.name = name  # self.name — атрибут объекта
        self.age = age

    def introduce(self):
        # self позволяет объекту "знать" свои данные
        return f"Меня зовут {self.name}, мне {self.age} лет"

p = Person("Алексей", 30)
print(p.introduce())  # Меня зовут Алексей, мне 30 лет
```

### Атрибуты и методы

- **Атрибуты** — это данные объекта (переменные внутри класса).
- **Методы** — это функции объекта (действия, которые объект может выполнять).

```python
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner      # Атрибут
        self.balance = balance  # Атрибут

    def deposit(self, amount):  # Метод
        self.balance += amount
        return f"Внесено {amount}. Баланс: {self.balance}"

    def withdraw(self, amount): # Метод
        if amount > self.balance:
            return "Недостаточно средств!"
        self.balance -= amount
        return f"Снято {amount}. Баланс: {self.balance}"

acc = BankAccount("Иван", 1000)
print(acc.deposit(500))   # Внесено 500. Баланс: 1500
print(acc.withdraw(200))  # Снято 200. Баланс: 1300
```

## Зачем это нужно

Без классов:
- Код был бы набором разрозненных функций и переменных
- Сложно было бы моделировать реальные сущности (пользователь, заказ, товар)
- Данные и функции, работающие с ними, были бы разбросаны по всему коду
- Невозможно было бы создавать библиотеки и фреймворки (FastAPI, Django, aiogram построены на ООП)

## Типичные ошибки

### Ошибка 1: Забытый self
```python
# ❌ Ошибка: self не передан в метод
class Dog:
    def __init__(name):  # Нет self!
        self.name = name

# ✅ Правильно:
class Dog:
    def __init__(self, name):
        self.name = name
```

### Ошибка 2: Изменяемые атрибуты класса
```python
# ❌ Опасно: список общий для всех объектов!
class Team:
    members = []  # Атрибут класса, а не объекта!

    def add(self, name):
        self.members.append(name)

team1 = Team()
team1.add("Иван")
team2 = Team()
print(team2.members)  # ['Иван'] — Ой! Иван попал в team2!

# ✅ Правильно: создавай список в __init__
class Team:
    def __init__(self):
        self.members = []  # У каждого объекта свой список

    def add(self, name):
        self.members.append(name)
```

### Ошибка 3: Обращение к атрибуту без self
```python
class Car:
    def __init__(self, brand):
        self.brand = brand

    def show(self):
        return brand  # ❌ NameError! Нужно self.brand

# ✅ Правильно:
    def show(self):
        return self.brand
```

## Итого

Класс — чертёж, объект — экземпляр.
`__init__` — конструктор, вызывается при создании объекта.
`self` — ссылка на текущий объект, обязательный первый параметр методов.
Атрибуты — данные объекта, методы — действия объекта.
Избегай изменяемых атрибутов класса (списки, словари) — создавай их в `__init__`.
ООП позволяет моделировать реальный мир и структурировать сложный код.
