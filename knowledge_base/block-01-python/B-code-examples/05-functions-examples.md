# Примеры кода: Функции

## Пример 1: Базовая функция с return

**Задача:** Написать функцию, которая считает площадь прямоугольника.

```python
# Определяем функцию через def
def rectangle_area(a, b):   # a и b — параметры (входные данные)
    area = a * b            # Локальная переменная — результат вычисления
    return area             # Возвращаем результат тому, кто вызвал функцию

# Вызываем функцию и сохраняем результат
result = rectangle_area(5, 3)   # a=5, b=3 → area=15 → return 15
print(result)               # 15

# Можно вызывать сколько угодно раз с разными аргументами
print(rectangle_area(10, 2))  # 20
print(rectangle_area(7, 7))   # 49
```

**Разбор:**
`def имя(параметры):` объявляет функцию. Тело пишется с отступом. `return` возвращает значение и завершает функцию. Без `return` функция возвращает `None`. Вызов функции — это имя со скобками и аргументами.

## Пример 2: Аргументы со значением по умолчанию

**Задача:** Функция приветствия с необязательным параметром.

```python
# greeting имеет значение по умолчанию "Привет"
def greet(name, greeting="Привет"):   # name обязателен, greeting — нет
    return f"{greeting}, {name}!"     # Формируем строку приветствия

# Вызов только с обязательным аргументом
print(greet("Алексей"))             # Привет, Алексей! (greeting по умолчанию)

# Вызов с обоими аргументами
print(greet("Мария", "Добрый день"))  # Добрый день, Мария!

# Именованный вызов — порядок не важен
print(greet(greeting="Здравствуй", name="Пётр"))  # Здравствуй, Пётр!
```

**Разбор:**
Параметры со значением по умолчанию (`greeting="Привет"`) можно не передавать при вызове. Они должны идти ПОСЛЕ обязательных параметров, иначе будет `SyntaxError`. Именованный вызов (`name=...`) делает код читаемее.

## Пример 3: *args и **kwargs

**Задача:** Функция, принимающая любое количество аргументов.

```python
# *args — собирает позиционные аргументы в кортеж
def sum_all(*args):         # args — кортеж всех переданных чисел
    print("Получил:", args)  # Показываем, что пришло
    return sum(args)        # sum() складывает все элементы кортежа

print(sum_all(1, 2, 3))     # Получил: (1, 2, 3) → 6
print(sum_all(10, 20))      # Получил: (10, 20) → 30
print(sum_all(5))           # Получил: (5,) → 5

print("---")

# **kwargs — собирает именованные аргументы в словарь
def print_info(**kwargs):   # kwargs — словарь всех именованных аргументов
    for key, value in kwargs.items():   # Перебираем пары ключ-значение
        print(f"{key} = {value}")

print_info(name="Аня", age=25, city="Москва")
# name = Аня
# age = 25
# city = Москва
```

**Разбор:**
`*args` (одна звёздочка) упаковывает позиционные аргументы в кортеж. `**kwargs` (две звёздочки) упаковывает именованные в словарь. Имена `args` и `kwargs` — соглашение, важны именно звёздочки. Это делает функции гибкими.

## Пример 4: Область видимости (scope)

**Задача:** Показать разницу между локальными и глобальными переменными.

```python
# Глобальная переменная — видна везде в файле
x = 10                      # Создаём в глобальной области

def my_function():          # Определяем функцию
    y = 20                  # Локальная переменная — видна ТОЛЬКО внутри функции
    print("Внутри: x =", x)  # 10 — глобальную читать МОЖНО
    print("Внутри: y =", y)  # 20 — локальную читать можно

my_function()               # Вызываем функцию
print("Снаружи: x =", x)    # 10 — глобальная доступна

# print("Снаружи: y =", y)  # NameError! y не существует снаружи функции
```

**Разбор:**
Локальные переменные (созданные внутри функции) живут только во время её выполнения. Снаружи их нет. Глобальные переменные видны внутри функции для чтения. Чтобы ИЗМЕНИТЬ глобальную внутри функции, нужно ключевое слово `global` (но это плохая практика — лучше передавать через аргументы и return).

## Пример 5: lambda-функции

**Задача:** Создать короткую анонимную функцию в одну строку.

```python
# Обычная функция
def double(x):              # Умножает на 2
    return x * 2

# Та же функция через lambda (анонимная, в одну строку)
double_lambda = lambda x: x * 2   # lambda параметры: выражение

print(double(5))            # 10
print(double_lambda(5))     # 10 — тот же результат

# lambda особенно удобна внутри sorted, map, filter
numbers = [3, 1, 4, 1, 5]

# Сортируем по модулю разницы с 3
sorted_nums = sorted(numbers, key=lambda x: abs(x - 3))
print(sorted_nums)          # [3, 4, 1, 1, 5] (3 ближе всех к 3)

# Фильтруем чётные через filter
evens = list(filter(lambda x: x % 2 == 0, [1, 2, 3, 4, 5, 6]))
print(evens)                # [2, 4, 6]
```

**Разбор:**
`lambda параметры: выражение` создаёт анонимную функцию без `def` и `return`. Выражение после двоеточия автоматически возвращается. lambda должна быть простой (одно выражение). Идеальна как аргумент для `sorted`, `map`, `filter`.

## Пример 6: Рекурсия (функция вызывает саму себя)

**Задача:** Посчитать факториал числа рекурсией.

```python
# Факториал n! = n * (n-1) * (n-2) * ... * 1
def factorial(n):           # Функция считает факториал числа n
    if n <= 1:              # БАЗОВЫЙ СЛУЧАЙ — условие выхода из рекурсии
        return 1            # 0! = 1 и 1! = 1
    return n * factorial(n - 1)   # РЕКУРСИВНЫЙ СЛУЧАЙ — функция вызывает себя

print(factorial(5))         # 5 * 4 * 3 * 2 * 1 = 120
# Раскрутка: 5 * factorial(4)
#            5 * 4 * factorial(3)
#            5 * 4 * 3 * factorial(2)
#            5 * 4 * 3 * 2 * factorial(1)
#            5 * 4 * 3 * 2 * 1 = 120
```

**Разбор:**
Рекурсия — когда функция вызывает саму себя. ОБЯЗАТЕЛЬНО нужен базовый случай (условие выхода), иначе рекурсия уйдёт в бесконечность и упадёт с `RecursionError`. Каждый вызов создаёт новый уровень в стеке. Для простых задач цикл часто эффективнее рекурсии.

## Частые ошибки новичков

| Ошибка | Пример | Решение |
|--------|--------|---------|
| Забыл return | функция печатает, но не возвращает | `return` вместо `print` внутри |
| Вызов без скобок | `print(my_func)` вместо `my_func()` | Добавь `()` для вызова |
| Нет self в методе | `def method():` в классе | `def method(self):` |
| Изменяемый default | `def f(lst=[])` | `def f(lst=None)` |
| Бесконечная рекурсия | нет базового случая | Добавь `if n <= 1: return` |
