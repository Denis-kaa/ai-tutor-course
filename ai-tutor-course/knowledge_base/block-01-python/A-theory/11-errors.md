# Обработка ошибок в Python

## Что такое ошибки и исключения

В программировании ошибки бывают двух типов:
1. **Синтаксические** — опечатки в коде (забытое двоеточие, неправильный отступ). Программа даже не запустится.
2. **Исключения (Exceptions)** — ошибки, которые возникают во время выполнения программы (деление на ноль, попытка открыть несуществующий файл, обращение к несуществующему ключу).

Исключения можно "ловить" и обрабатывать, чтобы программа не падала, а продолжала работать или корректно завершалась.

```python
try:
    result = 10 / 0
except ZeroDivisionError:
    print("На ноль делить нельзя!")
```

## Аналогии обработки ошибок

#### 🛡️ Аналогия 1: Подушка безопасности в машине
`try/except` — это как подушка безопасности. Ты едешь по дороге (выполняешь код в `try`). Если происходит авария (возникает ошибка), подушка срабатывает (выполняется `except`), и ты остаёшься жив (программа не падает). Без подушки — фатальный исход.

#### 🌂 Аналогия 2: Зонт в дождливую погоду
`try` — это прогулка без зонта. `except` — это зонт, который ты раскрываешь, если пошёл дождь (ошибка). Ты всё равно дойдёшь до цели, просто немного по-другому.

```python
try:
    walk_to_work()  # Идём пешком
except RainError:
    take_umbrella()  # Достаём зонт
    walk_to_work()   # Идём дальше
```

#### 🚧 Аналогия 3: Объездная дорога при ремонте
`try` — это основной маршрут. `except` — это объездная дорога, если на основном пути ремонт (ошибка). Ты всё равно доедешь до пункта назначения, просто другим путём.

#### 🩹 Аналогия 4: Аптечка первой помощи
`try/except` — как аптечка. Ты делаешь операцию (код в `try`). Если что-то пошло не так (ошибка), ты не бросаешь пациента, а оказываешь первую помощь (код в `except`), а потом продолжаешь.

#### 🔄 Аналогия 5: Запасной план на случай сбоя
`try` — это план А. `except` — это план Б. Если план А провалился, ты не паникуешь, а спокойно переключаешься на план Б.

## Полный синтаксис: try, except, else, finally

```python
try:
    # Код, который может вызвать ошибку
    number = int(input("Введите число: "))
    result = 100 / number
except ValueError:
    # Если ввели не число
    print("Это не число!")
except ZeroDivisionError:
    # Если ввели ноль
    print("На ноль делить нельзя!")
except Exception as e:
    # Ловим любую другую ошибку
    print(f"Произошла ошибка: {e}")
else:
    # Выполняется, если ошибок НЕ БЫЛО
    print(f"Результат: {result}")
finally:
    # Выполняется ВСЕГДА (была ошибка или нет)
    print("Операция завершена.")
```

## Основные типы исключений

| Исключение | Когда возникает | Пример |
|------------|-----------------|--------|
| `ValueError` | Неправильное значение | `int("abc")` |
| `TypeError` | Неправильный тип данных | `"2" + 2` |
| `KeyError` | Несуществующий ключ словаря | `{"a": 1}["b"]` |
| `IndexError` | Индекс вне диапазона списка | `[1, 2][5]` |
| `FileNotFoundError` | Файл не найден | `open("missing.txt")` |
| `ZeroDivisionError` | Деление на ноль | `10 / 0` |
| `AttributeError` | Несуществующий метод/атрибут | `"text".nonexistent()` |

## Зачем это нужно

Без обработки ошибок любая программа падала бы при первом же нестандартном вводе пользователя, отсутствии файла или потере сети. Обработка ошибок делает программы **устойчивыми** (robust) и **дружелюбными** к пользователю.

## Типичные ошибки

### Ошибка 1: "Голый" except (ловит всё подряд)
```python
# ❌ Плохо: ловит ВСЕ ошибки, включая опечатки в коде
try:
    do_something()
except:
    print("Что-то пошло не так")

# ✅ Хорошо: ловим только ожидаемые ошибки
try:
    do_something()
except (ValueError, TypeError) as e:
    print(f"Ошибка ввода: {e}")
```

### Ошибка 2: "Проглатывание" ошибки (ничего не делаем)
```python
# ❌ Плохо: ошибка произошла, но мы её игнорируем
try:
    risky_operation()
except Exception:
    pass  # Молча проглотили ошибку

# ✅ Хорошо: логируем или обрабатываем
try:
    risky_operation()
except Exception as e:
    print(f"Ошибка: {e}")
    # Или записываем в лог
```

### Ошибка 3: Забытый finally для закрытия ресурсов
```python
# ❌ Плохо: файл может остаться открытым при ошибке
f = open("data.txt")
data = f.read()
f.close()

# ✅ Хорошо: finally гарантирует закрытие
try:
    f = open("data.txt")
    data = f.read()
finally:
    f.close()

# ✅ Ещё лучше: используем контекстный менеджер (with)
with open("data.txt") as f:
    data = f.read()
# Файл закроется автоматически
```

## Итого

`try/except` — механизм обработки ошибок во время выполнения.
`else` — выполняется, если ошибок не было.
`finally` — выполняется всегда (для очистки ресурсов).
Лови только конкретные исключения, избегай "голого" `except`.
Используй `with` для работы с файлами и ресурсами.
