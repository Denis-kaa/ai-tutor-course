# Декораторы в Python

## Что такое декоратор

Декоратор — это функция, которая принимает другую функцию и возвращает её "улучшенную" версию, добавляя новое поведение, не меняя исходный код. Записывается через символ `@` перед определением функции.

Декораторы позволяют вынести повторяющуюся логику (логирование, проверка прав, замер времени) в одно место.

```python
def shout(func):
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        return result.upper() + "!!!"
    return wrapper

@shout  # Эквивалентно: greet = shout(greet)
def greet(name):
    return f"привет, {name}"

print(greet("мир"))  # ПРИВЕТ, МИР!!!
```

## Аналогии декораторов

#### 🎁 Аналогия 1: Подарочная упаковка
Декоратор — как упаковка подарка. Сам подарок (функция) не меняется, но снаружи добавляется красивая обёртка (новое поведение). Ты даришь то же самое, но с дополнительным эффектом.

#### 🖼️ Аналогия 2: Рамка для картины
Декоратор — как рамка вокруг картины. Картина (функция) остаётся той же, но рамка (декоратор) добавляет ей оформления: защиту, красоту, подпись. Картина не знает о рамке, но выглядит иначе.

#### 📱 Аналогия 3: Чехол для телефона
Декоратор — как чехол на смартфон. Телефон (функция) работает как прежде, но чехол добавляет защиту от ударов, подставку, держатель. Функционал расширен без вскрытия телефона.

#### 🛡️ Аналогия 4: Охранник на входе
Декоратор — как охранник перед дверью офиса. Прежде чем пустить тебя внутрь (вызвать функцию), охранник проверяет пропуск (права доступа). Если пропуска нет — внутрь не попадёшь, и сама функция даже не запустится.

```python
def require_login(func):
    def wrapper(user, *args, **kwargs):
        if not user.is_logged_in:
            return "Сначала войдите в систему!"
        return func(user, *args, **kwargs)
    return wrapper
```

#### 🗣️ Аналогия 5: Переводчик на переговорах
Декоратор — как переводчик. Ты говоришь на своём языке (функция возвращает результат), а переводчик (декоратор) преобразует это в нужный формат для собеседника, не меняя сути сказанного.

## Декоратор с аргументами

Иногда декоратору самому нужны параметры. Тогда он становится "фабрикой декораторов" — функцией, возвращающей декоратор.

```python
def repeat(times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)
def say_hi():
    print("Привет!")

say_hi()  # Напечатает "Привет!" три раза
```

## Встроенные декораторы

Python уже содержит полезные декораторы:

```python
class MyClass:
    @staticmethod       # Метод без доступа к self
    def add(a, b):
        return a + b

    @classmethod        # Метод с доступом к классу (cls)
    def create(cls):
        return cls()

    @property           # Превращает метод в атрибут
    def name(self):
        return self._name
```

## Зачем это нужно

Без декораторов повторяющаяся логика копировалась бы в каждую функцию:

```python
# ❌ Без декоратора — дублирование
def func_a():
    start = time.time()
    # ... логика ...
    print(time.time() - start)

def func_b():
    start = time.time()  # Снова то же самое!
    # ... логика ...
    print(time.time() - start)

# ✅ С декоратором — логика в одном месте
@timer
def func_a(): ...

@timer
def func_b(): ...
```

## Типичные ошибки

### Ошибка 1: Забыли *args, **kwargs
```python
# ❌ Декоратор сломается на функциях с аргументами
def bad_decorator(func):
    def wrapper():  # Нет параметров!
        return func()
    return wrapper

# ✅ Передаём любые аргументы
def good_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper
```

### Ошибка 2: Потеря имени функции
```python
# ✅ Используй functools.wraps, чтобы сохранить имя и документацию
from functools import wraps

def my_decorator(func):
    @wraps(func)  # Сохраняет __name__ и __doc__
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper
```

## Итого

Декоратор — функция-обёртка, записывается через `@decorator`.
Принимает функцию, возвращает улучшенную версию.
Внутри обычно определяет `wrapper(*args, **kwargs)`.
Декоратор с аргументами — функция, возвращающая декоратор.
Встроенные: `@staticmethod`, `@classmethod`, `@property`.
Используй `@functools.wraps`, чтобы не терять метаданные.
