# Условные конструкции в Python

## Что такое условия

Условные конструкции (if/elif/else) позволяют программе принимать решения. В зависимости от того, истинно условие или ложно, выполняется разный код.

Это основа "ветвления" — когда программа идёт по разным путям в зависимости от обстоятельств.

Базовый синтаксис:
\`\`\`python
if условие:
    # код, если условие истинно
elif другое_условие:
    # код, если первое ложно, но второе истинно
else:
    # код, если все условия ложны
\`\`\`

## Базовая конструкция if

\`\`\`python
age = 20

if age >= 18:
    print("Доступ разрешён")
\`\`\`

**Важные правила:**
1. После \`if\` всегда ставится двоеточие \`:\`
2. Тело \`if\` (код внутри) пишется с отступом (4 пробела)
3. Условие должно возвращать \`True\` или \`False\`

### Аналогии конструкции if

#### 🚦 Аналогия 1: Светофор

\`if\` — это светофор. Если горит зелёный (условие True) — идёшь. Если красный (False) — стоишь.

\`\`\`python
light = "green"

if light == "green":
    print("Можно идти")
# Если light != "green", программа просто идёт дальше, ничего не делая
\`\`\`

#### 🚪 Аналогия 2: Дверь с кодовым замком

\`if\` — это проверка кода на двери. Ввёл правильный код (условие True) — дверь открылась. Неправильный (False) — осталась закрытой.

\`\`\`python
password = "secret123"
input_password = input("Введите пароль: ")

if input_password == password:
    print("Дверь открыта!")
# Если пароль неверный — ничего не происходит
\`\`\`

#### 🎢 Аналогия 3: Ограничение по росту на аттракционе

\`if\` — это проверка роста перед входом на аттракцион. Рост >= 140 см — проходишь. Нет — не проходишь.

\`\`\`python
height = int(input("Твой рост в см: "))

if height >= 140:
    print("Добро пожаловать на аттракцион!")
\`\`\`

#### 📧 Аналогия 4: Фильтр спама в почте

\`if\` — как фильтр спама. Если письмо прошло проверку (условие True) — попадает во "Входящие". Если нет — в "Спам".

\`\`\`python
sender = "unknown@spam.com"
is_spam = "spam" in sender.lower()

if is_spam:
    print("Письмо перемещено в спам")
\`\`\`

#### 🎓 Аналогия 5: Проверка домашнего задания

\`if\` — как учитель, проверяющий задание. Если ответ правильный (True) — ставит "5". Если нет — ничего не делает (или ставит "2", но это уже \`else\`).

\`\`\`python
answer = 42
correct_answer = 42

if answer == correct_answer:
    print("Молодец, правильный ответ!")
\`\`\`

## Конструкция if-else

Добавляем "иначе" — что делать, если условие ложно:

\`\`\`python
age = 16

if age >= 18:
    print("Доступ разрешён")
else:
    print("Доступ запрещён")
\`\`\`

**Важно:** \`else\` выполняется **только если** условие \`if\` ложно. Всегда выполняется либо \`if\`, либо \`else\` — что-то одно.

### Аналогии if-else

#### 🎭 Аналогия 1: Две роли актёра

\`if-else\` — как у актёра две роли. Если его вызвали на сцену (условие True) — играет роль А. Если не вызвали (False) — играет роль Б. Третьего не дано.

\`\`\`python
is_called = False

if is_called:
    print("Играю главную роль")
else:
    print("Играю второстепенную")
\`\`\`

#### 🛤️ Аналогия 2: Развилка на дороге

\`if-else\` — это развилка. Налево (True) — в город. Направо (False) — в деревню. Ты обязательно выберешь один путь.

\`\`\`python
has_money = True

if has_money:
    print("Еду в город")
else:
    print("Иду в деревню")
\`\`\`

#### 💡 Аналогия 3: Выключатель света

\`if-else\` — как выключатель. Включён (True) — светло. Выключен (False) — темно. Промежуточных состояний нет.

\`\`\`python
is_daytime = False

if is_daytime:
    print("Свет выключен")
else:
    print("Свет включён")
\`\`\`

#### 🎰 Аналогия 4: Выигрыш в лотерею

\`if-else\` — как проверка лотерейного билета. Выигрыш есть (True) — радуешься. Нет (False) — расстраиваешься.

\`\`\`python
ticket_number = 123
winning_number = 456

if ticket_number == winning_number:
    print("Ты выиграл!")
else:
    print("Не повезло, попробуй ещё раз")
\`\`\`

#### 🍽️ Аналогия 5: Заказ в ресторане

\`if-else\` — как выбор блюда. Если есть стейк (True) — берёшь стейк. Если нет (False) — берёшь салат.

\`\`\`python
has_steak = False

if has_steak:
    print("Беру стейк")
else:
    print("Тогда салат")
\`\`\`

## Конструкция if-elif-else

Когда нужно проверить несколько условий:

\`\`\`python
score = 85

if score >= 90:
    print("Оценка: 5 (отлично)")
elif score >= 75:
    print("Оценка: 4 (хорошо)")
elif score >= 60:
    print("Оценка: 3 (удовлетворительно)")
else:
    print("Оценка: 2 (неудовлетворительно)")
\`\`\`

**Важно:** Python проверяет условия **сверху вниз** и останавливается на первом истинном. Даже если несколько условий истинны, выполнится только первый блок.

### Аналогии if-elif-else

#### 🏆 Аналогия 1: Медали на соревнованиях

\`if-elif-else\` — как награждение. Первое место (>= 90) — золото. Второе (>= 75) — серебро. Третье (>= 60) — бронза. Иначе — утешительный приз.

\`\`\`python
points = 88

if points >= 90:
    print("🥇 Золотая медаль")
elif points >= 75:
    print("🥈 Серебряная медаль")
elif points >= 60:
    print("🥉 Бронзовая медаль")
else:
    print("Утешительный приз")
\`\`\`

#### 🎫 Аналогия 2: Возрастные категории в кино

\`if-elif-else\` — как цены на билеты по возрасту. До 7 лет — бесплатно. 7-17 — детский. 18-64 — взрослый. 65+ — пенсионер.

\`\`\`python
age = 25

if age < 7:
    price = 0
elif age < 18:
    price = 200
elif age < 65:
    price = 400
else:
    price = 250

print(f"Цена билета: {price} руб.")
\`\`\`

#### 🚦 Аналогия 3: Светофор с жёлтым светом

\`if-elif-else\` — как светофор с тремя цветами. Зелёный (if) — иди. Жёлтый (elif) — готовься. Красный (else) — стой.

\`\`\`python
light = "yellow"

if light == "green":
    print("Иди")
elif light == "yellow":
    print("Приготовься")
else:  # красный
    print("Стой")
\`\`\`

#### 📦 Аналогия 4: Размеры одежды

\`if-elif-else\` — как подбор размера. До 44 — XS. 44-46 — S. 48-50 — M. 52-54 — L. Больше — XL.

\`\`\`python
size = 48

if size < 44:
    label = "XS"
elif size <= 46:
    label = "S"
elif size <= 50:
    label = "M"
elif size <= 54:
    label = "L"
else:
    label = "XL"

print(f"Твой размер: {label}")
\`\`\`

#### 🎓 Аналогия 5: Уровни доступа

\`if-elif-else\` — как уровни доступа в компании. Стажёр (if) — только чтение. Разработчик (elif) — чтение и запись. Менеджер (elif) — всё. Директор (else) — полный доступ.

\`\`\`python
role = "developer"

if role == "intern":
    access = "read only"
elif role == "developer":
    access = "read and write"
elif role == "manager":
    access = "full"
else:
    access = "admin"
\`\`\`

## Вложенные условия

Условия можно вкладывать друг в друга:

\`\`\`python
age = 25
has_license = True

if age >= 18:
    if has_license:
        print("Можешь водить")
    else:
        print("Нужно получить права")
else:
    print("Слишком молод")
\`\`\`

**Но!** Глубокая вложенность плохо читается. Лучше использовать логические операторы:

\`\`\`python
if age >= 18 and has_license:
    print("Можешь водить")
elif age >= 18 and not has_license:
    print("Нужно получить права")
else:
    print("Слишком молод")
\`\`\`

### Аналогия вложенных условий

#### 🏠 Аналогия 1: Комнаты в доме

Вложенные условия — как комнаты в доме. Чтобы попасть на кухню (внутреннее условие), нужно сначала войти в дом (внешнее условие).

\`\`\`python
is_home = True
is_hungry = True

if is_home:
    if is_hungry:
        print("Иду на кухню")
    else:
        print("Отдыхаю в гостиной")
else:
    print("Меня нет дома")
\`\`\`

#### 📁 Аналогия 2: Папки в папках

Вложенные условия — как папки внутри папок. Чтобы открыть файл, нужно сначала открыть папку, потом подпапку.

\`\`\`python
has_folder = True
has_subfolder = True

if has_folder:
    if has_subfolder:
        print("Открываю файл")
    else:
        print("Подпапки нет")
else:
    print("Папки нет")
\`\`\`

## Тернарный оператор

Короткая запись if-else в одну строку:

\`\`\`python
age = 20
status = "взрослый" if age >= 18 else "ребёнок"
print(status)  # взрослый
\`\`\`

**Синтаксис:** \`результат_если_истина if условие else результат_если_ложь\`

### Аналогия тернарного оператора

#### 🎲 Аналогия 1: Монетка

Тернарный оператор — как подбрасывание монетки. Орёл (True) — одно действие. Решка (False) — другое. Всё в одном движении.

\`\`\`python
import random
coin = random.choice(["орёл", "решка"])
result = "Выиграл" if coin == "орёл" else "Проиграл"
\`\`\`

#### 🎯 Аналогия 2: Быстрый выбор

Тернарный оператор — как быстрый выбор в магазине. Видишь товар — берёшь (True). Не видишь — не берёшь (False). Без долгих раздумий.

\`\`\`python
in_stock = True
action = "Купить" if in_stock else "Заказать"
\`\`\`

## Типичные ошибки

### Ошибка 1: \`=\` вместо \`==\`

\`\`\`python
# ❌ Неправильно:
if x = 5:  # Это присваивание, а не сравнение!
    print("x равен 5")

# ✅ Правильно:
if x == 5:
    print("x равен 5")
\`\`\`

### Ошибка 2: Забытое двоеточие

\`\`\`python
# ❌ Неправильно:
if x > 5
    print("Больше 5")

# ✅ Правильно:
if x > 5:
    print("Больше 5")
\`\`\`

### Ошибка 3: Неправильные отступы

\`\`\`python
# ❌ Неправильно:
if x > 5:
print("Больше 5")  # Нет отступа!

# ✅ Правильно:
if x > 5:
    print("Больше 5")  # 4 пробела отступа
\`\`\`

### Ошибка 4: Порядок elif

\`\`\`python
# ❌ Неправильно:
score = 85
if score >= 60:
    print("Тройка")
elif score >= 90:  # Никогда не выполнится!
    print("Пятёрка")

# ✅ Правильно: от большего к меньшему
if score >= 90:
    print("Пятёрка")
elif score >= 60:
    print("Тройка")
\`\`\`

## Зачем это нужно

Без условий:
- Программа не могла бы принимать решения
- Все пользователи получали бы одинаковый результат
- Невозможно было бы проверять пароли, права доступа, валидность данных
- Любая логика сводилась бы к линейному выполнению

## Что было бы, если бы условий не существовало

Представь, что ты пишешь программу для банкомата. Без условий:

\`\`\`python
# Без условий — всем выдаём деньги, даже если на счету 0
balance = 0
withdraw = 1000
balance -= withdraw
print("Выдано 1000 руб.")  # Даже если денег нет!
\`\`\`

**С условиями:**
\`\`\`python
balance = 0
withdraw = 1000

if balance >= withdraw:
    balance -= withdraw
    print("Выдано 1000 руб.")
else:
    print("Недостаточно средств")
\`\`\`

## Связь с другими концепциями

Условия тесно связаны с:
- **Операторами сравнения** — для формирования условий
- **Логическими операторами** — для сложных условий
- **Циклами** — для проверки условия на каждой итерации
- **Функциями** — для возврата разных значений в зависимости от условий

Условия — это "мозг" программы. Без них код был бы глупым исполнителем, не способным адаптироваться к обстоятельствам.

## Итого

\`if\` — если условие истинно, выполнить код.
\`elif\` — если предыдущее условие ложно, проверить это.
\`else\` — если все условия ложны, выполнить этот код.

Правила:
- После \`if\`, \`elif\`, \`else\` — двоеточие \`:\`
- Тело — с отступом (4 пробела)
- Python проверяет условия сверху вниз и останавливается на первом истинном
- \`else\` всегда последний и выполняется, если ничего выше не сработало

Тернарный оператор: \`результат if условие else результат\` — короткая запись if-else.
