# Модули и импорты в Python

## Что такое модуль

Модуль — это файл с расширением `.py`, содержащий функции, классы и переменные, которые можно использовать в других программах. Модули позволяют:
- Не писать один и тот же код заново
- Разбивать большую программу на маленькие файлы
- Использовать готовые решения из стандартной библиотеки и сторонних пакетов

```python
# math.py — это модуль
import math
print(math.sqrt(16))  # 4.0
```

## Аналогии модулей

#### 🧰 Аналогия 1: Ящик с инструментами
Модуль — как ящик с инструментами. Тебе не нужно изобретать молоток или отвёртку каждый раз. Ты просто берёшь готовый инструмент из ящика (импортируешь модуль) и используешь его.

```python
import os  # Берём "ящик" с инструментами для работы с ОС
print(os.getcwd())  # Используем инструмент "узнать текущую папку"
```

#### 📚 Аналогия 2: Библиотека с книгами
Модуль — как книга в библиотеке. В книге "Математика" есть главы (функции): `sqrt`, `sin`, `cos`. Ты берёшь книгу (импортируешь модуль) и открываешь нужную главу.

```python
from math import sqrt, sin  # Берём конкретные главы из книги
print(sqrt(25))  # 5.0
```

#### 🍳 Аналогия 3: Книга рецептов
Модуль — как книга рецептов. Тебе не нужно помнить все рецепты наизусть. Ты импортируешь книгу и вызываешь нужный рецепт по имени.

```python
import recipes  # Импортируем книгу рецептов
recipes.make_pizza()  # Готовим пиццу по рецепту
```

#### 🏗️ Аналогия 4: Строительные блоки / Конструктор
Модуль — как набор деталей конструктора. У тебя есть готовые блоки (функции), из которых ты собираешь свою программу, как дом из кирпичиков.

#### 📦 Аналогия 5: Коробка с запчастями
Модуль — как коробка с запчастями для машины. Тебе не нужно вытачивать каждую шестерёнку вручную. Ты берёшь готовую деталь из коробки и устанавливаешь.

## Способы импорта

```python
# 1. Импорт всего модуля
import math
print(math.pi)  # Обращаемся через точку

# 2. Импорт конкретных функций
from math import sqrt, pow
print(sqrt(9))  # Обращаемся напрямую, без math.

# 3. Импорт с псевдонимом (alias)
import numpy as np  # Короткое имя для длинного модуля
print(np.array([1, 2, 3]))

# 4. Импорт всего содержимого (не рекомендуется!)
from math import *  # Загрязняет пространство имён
```

## Стандартная библиотека Python

В Python "встроены" десятки модулей, которые не нужно устанавливать:

| Модуль | Назначение | Пример |
|--------|-----------|--------|
| `math` | Математические функции | `math.sqrt(16)` |
| `random` | Случайные числа | `random.randint(1, 10)` |
| `datetime` | Работа с датами | `datetime.now()` |
| `os` | Работа с ОС и файлами | `os.listdir()` |
| `json` | Работа с JSON | `json.loads(data)` |
| `sys` | Системные параметры | `sys.argv` |
| `re` | Регулярные выражения | `re.search(pattern, text)` |

## Сторонние модули и pip

Для установки сторонних библиотек используется менеджер пакетов `pip`:

```bash
# Установка библиотеки
pip install requests

# Удаление
pip uninstall requests

# Список установленных
pip list
```

```python
# После установки можно импортировать
import requests
response = requests.get("https://api.github.com")
print(response.status_code)
```

## Создание собственного модуля

Любой `.py` файл — это модуль. Создадим файл `helpers.py`:

```python
# helpers.py
def greet(name):
    return f"Привет, {name}!"

def add(a, b):
    return a + b
```

Используем его в другом файле:

```python
# main.py
import helpers

print(helpers.greet("Алексей"))  # Привет, Алексей!
print(helpers.add(5, 3))         # 8
```

## Магическая переменная __name__

Когда файл запускается напрямую, `__name__` равно `"__main__"`. Когда импортируется — имени модуля.

```python
# helpers.py
def greet(name):
    return f"Привет, {name}!"

if __name__ == "__main__":
    # Этот код выполнится только при запуске helpers.py напрямую
    print(greet("Тест"))
    # Но НЕ выполнится при импорте из другого файла
```

## Зачем это нужно

Модули — это основа организации кода. Без них:
- Весь код был бы в одном огромном файле
- Приходилось бы копировать функции из проекта в проект
- Невозможно было бы использовать чужие библиотеки
- Код было бы невозможно поддерживать и тестировать

## Типичные ошибки

### Ошибка 1: ModuleNotFoundError
```python
# ❌ Модуль не установлен или опечатка в имени
import reqests  # Опечатка!

# ✅ Проверь имя и установи модуль
# pip install requests
import requests
```

### Ошибка 2: Циклический импорт
```python
# ❌ a.py импортирует b.py, а b.py импортирует a.py
# a.py
import b

# b.py
import a  # Цикл! Ошибка или непредсказуемое поведение

# ✅ Решение: перенести общий код в третий модуль c.py
```

### Ошибка 3: Импорт всего (*)
```python
# ❌ Загрязняет пространство имён, непонятно откуда функции
from math import *

# ✅ Явный импорт
from math import sqrt, pi
```

## Итого

Модуль — файл `.py` с функциями и классами.
Импорт: `import module`, `from module import func`, `import module as alias`.
Стандартная библиотека: `math`, `random`, `datetime`, `os`, `json`.
Сторонние модули устанавливаются через `pip install`.
`if __name__ == "__main__":` — код выполняется только при прямом запуске файла.
